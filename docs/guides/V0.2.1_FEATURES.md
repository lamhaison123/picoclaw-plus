# v0.2.1 Features Guide

Complete guide to all v0.2.1 features integrated into PicoClaw.

## Overview

PicoClaw has successfully integrated all 10 features from v0.2.1, bringing significant improvements in reliability, performance, and flexibility.

## Features

### 1. JSONL Memory Store
**Status**: ✅ Production Ready  
**Impact**: Critical - Crash-safe storage

Crash-safe append-only session storage that prevents data loss.

**Benefits**:
- No data loss on power failure or crashes
- Automatic fsync for durability
- Per-session file locking
- Auto-migration from JSON

**Configuration**:
```json
{
  "session": {
    "storage_backend": "jsonl",
    "auto_migrate": true
  }
}
```

**Learn More**: See [JSONL Storage Guide](../reference/JSONL_STORAGE.md)

---

### 2. Vision/Image Support
**Status**: ✅ Production Ready  
**Impact**: High - Multi-modal AI

Send images to AI models for analysis and understanding.

**Supported Models**:
- OpenAI: GPT-4V, GPT-4o
- Anthropic: Claude 3+

**Usage**:
```bash
picoclaw agent
> Describe this image: /path/to/image.jpg
> Analyze media://chart.png
```

**Learn More**: See [Vision Support Guide](../guides/VISION_SUPPORT.md)

---

### 3. Parallel Tool Execution
**Status**: ✅ Production Ready  
**Impact**: High - 2x faster

Tools execute in parallel automatically for better performance.

**Benefits**:
- 2x faster tool calls
- Automatic parallelization
- Ordered result collection
- No configuration needed

---

### 4. Model Routing
**Status**: ✅ Production Ready  
**Impact**: High - Cost optimization

Automatically route queries to appropriate models based on complexity.

**Configuration**:
```json
{
  "routing": {
    "enabled": true,
    "tiers": [
      {"name": "cheap", "models": ["gpt-4o-mini"]},
      {"name": "expensive", "models": ["gpt-5.2"]}
    ]
  }
}
```

**Learn More**: See [Model Routing Guide](../guides/MODEL_ROUTING.md)

---

### 5. Environment Configuration
**Status**: ✅ Production Ready  
**Impact**: Medium - Flexible configuration

Configure PicoClaw using .env files.

**Precedence**:
```
Environment Variables > .env.local > .env > config.json > defaults
```

**Example .env**:
```bash
PICOCLAW_ROUTING_ENABLED=true
PICOCLAW_SESSION_STORAGE_BACKEND=jsonl
OPENAI_API_KEY=sk-...
```

**Learn More**: See [Configuration Guide](../reference/CONFIGURATION.md)

## Quick Start

### Enable All Features
```json
{
  "routing": {
    "enabled": true,
    "tiers": [
      {"name": "cheap", "models": ["gpt-4o-mini"]},
      {"name": "expensive", "models": ["gpt-5.2"]}
    ]
  },
  "session": {
    "storage_backend": "jsonl",
    "auto_migrate": true,
    "summarization_message_threshold": 20,
    "summarization_token_percent": 0.75
  },
  "tools": {
    "file_tools_enabled": true,
    "shell_tools_enabled": true,
    "web_tools_enabled": true
  },
  "agents": {
    "defaults": {
      "model": "gpt-4o",
      "max_media_size": 20971520
    }
  }
}
```

### Using .env
```bash
# Create .env file
cat > .env << EOF
PICOCLAW_ROUTING_ENABLED=true
PICOCLAW_SESSION_STORAGE_BACKEND=jsonl
PICOCLAW_TOOLS_WEB_ENABLED=true
OPENAI_API_KEY=sk-...
EOF

# Run
picoclaw agent
```
